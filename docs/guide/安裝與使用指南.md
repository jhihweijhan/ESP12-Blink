# Mochi-Metrics 圖文安裝與使用指南

這份指南給第一次接觸專案的使用者。
你可以照順序一步一步做，不需要先了解全部細節。

## 目錄

1. 安裝前準備
2. 硬體確認
3. 燒錄韌體
4. 安裝 Sender
5. 首次設定（設定精靈）
6. 驗證是否成功
7. 常見故障排除

## 1) 安裝前準備

### 必備

- 一台 ESP12 顯示模組（已接好電源與螢幕）
- 一台 Linux/Ubuntu 電腦（建議）
- 可連線的 MQTT broker
- USB 線（燒錄用）

### 軟體

- PlatformIO（用來編譯/燒錄韌體）
- Go 1.22+（用來編譯 Sender）— 或使用預編譯版本
- Git

## 2) 硬體確認（型號）

請確認你的組件型號對應如下：

| 元件 | 型號 |
|---|---|
| LCD | `BL-A54038P-02` |
| Wi-Fi 模組 | `ESP-12F` |

## 3) 燒錄韌體

在專案根目錄執行：

```bash
cd apps/firmware
~/.platformio/penv/bin/pio run
~/.platformio/penv/bin/pio run -t upload
~/.platformio/penv/bin/pio device monitor --baud 115200
```

如果上傳序列埠錯誤，請先檢查 `apps/firmware/platformio.ini` 的 `upload_port`。

## 4) 安裝 Sender

Sender 負責收集電腦的 CPU、RAM、GPU 等指標並透過 MQTT 發送給 ESP。

### Go Sender 一鍵安裝（推薦）

各平台提供一鍵安裝腳本，自動編譯、安裝、註冊系統服務：

| 平台 | 安裝指令 | 服務管理 |
|------|---------|---------|
| Linux | `bash apps/sender/go/install.sh --mqtt-host=<broker-ip>` | systemd user service |
| macOS | `bash apps/sender/go/install-macos.sh --mqtt-host=<broker-ip>` | launchd agent |
| Windows | `powershell apps/sender/go/install.ps1 -MqttHost <broker-ip>` (**以系統管理員執行**) | Scheduled Task |

安裝腳本會自動：
- 編譯 Go Sender（需要 Go 1.22+）
- 安裝為系統服務（開機自動啟動）
- 設定 MQTT 連線參數

卸載：各腳本支援 `--uninstall`（Windows: `-Uninstall`）參數。

### Python Sender（替代方案）

```bash
cd apps/sender/python
pip install -e .
python sender_v2.py
```

需設定環境變數：`MQTT_HOST`、`MQTT_PORT`（預設 1883）。

### Docker Compose（進階用戶）

如果偏好容器化部署：

```bash
cd apps/sender/python
chmod +x senderctl.sh
./senderctl.sh quickstart-compose
```

## 5) 首次設定（設定精靈）

ESP 首次開機會自動進入 AP 模式，螢幕上會顯示 QR code 和連線資訊。

### 連接設定精靈

1. 手機或電腦連接 ESP 的 AP 熱點
2. 開啟瀏覽器前往 `http://192.168.4.1/setup`
3. 按照精靈步驟依序完成：

| 步驟 | 說明 |
|------|------|
| Step 1: WiFi | 掃描並選擇 WiFi 網路，輸入密碼 |
| Step 2: MQTT | 輸入 broker IP 和連接埠，測試連線 |
| Step 3: Sender | 複製對應平台的安裝指令到電腦執行 |
| Step 4: 完成 | 確認設定成功，自動跳轉監控頁 |

### 手動設定（非首次）

如果需要修改設定，可直接訪問：
- `http://<esp-ip>/setup` — 重新執行設定精靈
- `http://<esp-ip>/monitor` — 修改 MQTT 和裝置設定
- `http://<esp-ip>/status` — 查看即時系統狀態

## 6) 驗證是否成功

成功時通常會看到：

- 螢幕底部顯示 `MQTT OK`
- CPU / RAM / GPU 資料持續更新
- API 狀態可回應（範例）：

```bash
curl http://<esp-ip>/api/v2/status
```

## 7) 常見故障排除

### 問題 A：螢幕顯示 `MQTT --`

- 檢查 broker 位址是否可從 ESP 連到
- 檢查 MQTT 帳密是否正確
- 查看 Sender 日誌：`./senderctl.sh compose-logs`

### 問題 B：有溫度但 GPU 使用率常是 `0%`

這通常代表 GPU 當下空閒，不一定是錯誤。
可在主機端執行 GPU 工作（例如影片編碼）觀察是否上升。

### 問題 C：Sender 執行中，但 ESP 沒收到資料

- 確認 Sender 的 hostname 與 topic 正確
- 確認 ESP 的訂閱 topic 為 `sys/agents/+/metrics/v2`
- 檢查 broker 是否有 ACL 限制
- 檢查 Sender 服務狀態：Linux `systemctl --user status mochi-sender`、macOS `launchctl list | grep mochi`
